JS中的继承机制和其他语言是不一样的

# 原型继承

原型继承：

1.并不会把父类中的方法、属性等克隆一份给子类，而是建立了子类和父类的原型链查找机制

2.重定向子类的原型后，默认丢失了原本的constructor属性（或者原本在原型上设置的属性和方法）

3.子类或者子类的实例可以基于原型链修改父类的属性方法

4.会把父类私有的属性、方法做为子类公有的属性方法继承过来

function A() {
  this.name = 'f'
}

A.prototype.say = function () {
  console.log('我要喝水')
}

function B() {
  this.age = 10
}

B.prototype.eat = function () {
  console.log('吃饭')
}

console.log(A.prototype.constructor === B.prototype.constructor) // false
console.log(A.prototype.constructor)
/* 
ƒ A() {
  this.name = 'f'
}
  */

console.log(B.prototype.constructor)
/* 
ƒ B() {
  this.age = 10
}
  */

B.prototype = new A()

const b = new B
b.say()

// b.eat() // b.eat is not a function

console.log(b.name); // f
b.name = 'y'

const a = new A
console.log(a.name) // f
a.say() // 我要喝水

console.log(A.prototype.constructor);
/* 
ƒ A() {
  this.name = 'f'
}
  */

//  B的构造函数被改成了A
console.log(B.prototype.constructor);
/* 
ƒ A() {
  this.name = 'f'
}
  */

# call继承

把父类当做普通函数执行，让其执行的时候方法中的this变为子类的实例即可

function A() {
  this.name = 'f'
}
A.prototype.say = function() {
  console.log('say')
}
function B() {
  A.call(this)
  this.age = 10
}
const b = new B

// 可以访问A中的name属性
console.log(b.name) // f

// 但是无法访问A原型上的方法say
b.say() // b.say is not a function

call继承的特点：

1.只能继承父类中的私有属性（继承的私有属性赋值给子类实例的私有属性）而且是类似拷贝，不是链式查找

2.因为只是把父类当做普通的方法执行，所以父类原型上的公有属性方法无法被继承过来

# 寄生组合继承

通过call继承和变异版的原型继承共同完成的，通过call实现私有继承，通过原型实现公有继承

function A() {
  this.name = 'f'
}
A.prototype.say = function() {
  console.log('say')
}
function B() {
  A.call(this)
  this.age = 10
}

B.prototype = Object.create(A.prototype)
B.prototype.constructor = B

const b = new B
console.log(b.name) // f
b.say() // say

# class实现继承

class A {
  constructor() {
    this.x = 100
  }
  getX() {
    console.log(this.x)
  }
}

class B extends A {
  constructor() {
    super()
    this.y = 200
  }
  getY() {
    console.log(this.x)
  }
}

const b = new B
b.getY() // 100
