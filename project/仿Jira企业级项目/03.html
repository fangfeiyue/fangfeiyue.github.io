# 为什么我们需要TS - 真实场景学习 TS 的必要性

使用JS的时候。比如一些非关键字的拼写错误、参数传递类型错误等，只有在项目运行的时候才能发现。我们希望在静态代码的时候就能发现这些错误，并加以修改，这样可以很好的提高编码效率。

# 将项目列表页面JS改造成TS，增强类型，减少Bug

src/screens/project-list/index.tsx

import { useEffect, useState } from "react";
import { List } from "./list";
import { SearchPannel } from "./search-pannel";
import { toQuery, cleanObject } from 'utils'
import { useMount } from "hook/use-mount";
import { useDebounce } from "hook/use-debounce";

const apiUrl = process.env.REACT_APP_API_URL

export const ProjectListScreen = () => {
  const [list , setList] = useState([])
  const [users, setUsers] = useState([])
  const [param, setParam] = useState({
    name: '',
    personId: ''
  })

  const debouncedParam = useDebounce(param , 200)
  useEffect(() => {
    fetch(`${apiUrl}/projects?${toQuery(cleanObject(debouncedParam))}`).then(async res => {
      if (res.ok) {
        setList(await res.json())
      }
    })
  }, [debouncedParam])

  useMount(() => {
    fetch(`${apiUrl}/users`).then(async res => {
      if (res.ok) {
        setUsers(await res.json())
      }
    })
  })

  return (
    <div>
      <SearchPannel param={param} setParam={setParam} users={users}/>
      <List list={list} users={users}/>
    </div>
  );
};

src/utils/index.ts

// export const isFalsy = (value: any): boolean => (value === 0 ? false : !value);
const isFalsy: (value: unknown) => boolean = (value) => (value === 0 ? false : !value);

export const cleanObject = (object: object) => {
  const obj = { ...object };

  Object.keys(obj).forEach((key) => {
    // @ts-ignore
    const value = obj[key];
    // @ts-ignore
    if (isFalsy(value)) delete obj[key];
  });

  return obj;
};

export const toQuery = (obj: any) => {
  if (isFalsy(obj) || typeof obj !== "object") return "";

  let str = "";
  for (let key in obj) {
    str += "&" + key + "=" + obj[key];
  }
  return str.slice(1);
};

src/hook/use-debounce.ts

import { useEffect, useState } from "react";

export const useDebounce = (value: any, delay?: number) => {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay);
    // 返回的这个函数在上一次的useEffect运行完后再运行，一般可以用于一些清理的工作
    return () => clearTimeout(timer);
  }, [value, delay]);

  return debouncedValue;
};

TS是强力型版的JS，当我们在代码中定义变量的时候，TS允许我们在定义的同时指定类型，这样使用者在使用不当的时候就会被及时报错提醒。

经常用帖子的感受。比起原来的js，ts带来了完全不一样的体验，bug大大减少了，编译器提示快了，代码更易读了，开发速度快了。

# unknown

# 学习泛型，用泛型增强useDebounce类型灵活性

function getValue(value: number): number {
  return value
}

上面我们定义的一个 getValue 函数，它的参数和返回值都是 number 类型。如果我们想让这个函数支持更多种数据类型该怎么办呢？比如说我们希望传入 string 返回 string ，这个时候我们就使用到了泛型。先看效果：

function getValue<T>(value:T): T {
  return value
}

console.log(getValue(1)) // 1
console.log(getValue('a')) // 'a'
console.log(getValue(true)) // true
console.log(getValue({})) // {}

我们给 getValue 函数添加了类型变量 T , T 可以帮我们捕获用户传入的类型（如： string）。后面我们就可以使用这个类型来指定函数参数及返回值的类型，大大方便了，代码的重用性。

下面我们把之前的 useDebounce 使用泛型改造下，以便它能支持更多的数据类型。

import { useEffect, useState } from "react";

export const useDebounce = <T>(value: T, delay?: number) => {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay);
    // 返回的这个函数在上一次的useEffect运行完后再运行，一般可以用于一些清理的工作
    return () => clearTimeout(timer);
  }, [value, delay]);

  return debouncedValue;
};

# 用 Hook + TS + TS泛型实现useArray

src/hook/use-array.ts

import { useState } from "react"

export const useArray = <T>(initialArray: T[]) => {
  const [value, setValue] = useState(initialArray)
  return { 
    value,
    clear: () => setValue([]), 
    removeIndex: (index: number) => {
      const arr = [...value]
      arr.splice(index, 1)
      setValue(arr)
    }, 
    add: (item: T) => {
      setValue([
        ...value,
        item
      ])
    }
  }
}
