# JWT理论介绍

关于 JWT 推荐阅读阮一峰老师的 JSON Web Token 入门教程 (opens new window) 和 五分钟带你了解啥是JWT (opens new window)，这里仅做简要介绍。

JWT 是一种跨域认证解决方案，全称是 JSON Web Token。

JWT 解决了什么问题？

  • 数据传输简单、高效
  • JWT 会生成签名，保证传输安全
  • JWT 具有时效性
  • JWT 更高效利用集群做好单点登录；

# JWT 原理

JWT 的原理是：服务器认证后，生成一个 JSON 对象，发回给客户端，后续通过这个 JSON 对象进行认定身份。为了保证安全会给这个对象加上签名。

{
  "name": "f",
  "age": 18
}

# JWT 数据结构

JWT 由三部分组成，分别是：

  • Header（头部）
  • Payload（负载）
  • Signature（签名）

JWT 的数据结构大体如下：

Header 部分是一个 JSON 对象，描述 JWT 的元数据：

{
  "alg": "HS256",
  "typ": "JWT"
}

Payload 部分也是一个 JSON 对象，用来存放实际需要传输的数据，官方规定的有如下字段：

  • iss (issuer)：签发人
  • exp (expiration time)：过期时间
  • sub (subject)：主题
  • aud (audience)：受众
  • nbf (Not Before)：生效时间
  • iat (Issued At)：签发时间
  • jti (JWT ID)：编号 我们还可以传输自己定义的字段，如：
{
  "name": "f"
}

JWT 使用方式

# 通过jwt生成token

我们可以通过 jsonwebtoken (opens new window) 来生成 token 。

安装

npm install jsonwebtoken -S



 













 
 
 
 


 
 










// routes/users.js

const router = require('koa-router')();
const User = require('./../models/userSchema');
const Util = require('../utils/utils');
var jwt = require('jsonwebtoken');
router.prefix('/users');

router.post('/login', async (ctx) => {
	try {
		const { userName, password } = ctx.request.body;
    console.log(userName, password)
		const res = await User.findOne({
			userName,
			userPwd: password
		});

    const data = res._doc;
    const token = jwt.sign({
      data: data
    }, 'fang', { expiresIn: 20 });

		if (res) {
      data.token = token;
			ctx.body = Util.success(data);
		} else {
			ctx.body = Util.fail('账号或密码不正确');
		}
	} catch (err) {
		ctx.body = Util.fail(err);
	}
});

module.exports = router;

# Token拦截

安装 koa-jwt 用于验证 JWT token：

npm install koa-jwt

简单使用：

// app.js

// ...
const jwt = require('koa-jwt')
// ...

app.use(jwt({ secret: 'fang' }).unless({
  path: [/^\/api\/users\/login/]
}));
// ...

细心的同学可能看到了上面代码中的 unless，这个是干啥的呢，这个是用于排除 token 校验的，有些接口不需要 token 校验，可以在 unless 里排除。

当校验失败，接口默认会返回错误信息：

如果你不想使用这个自定义信息，可以通过中间件的方式重新定义返回的信息：

// app.js

// ...
const jwt = require('koa-jwt')
// ...

app.use(async function(ctx, next) {
  await next().catch((err) => {
    if (err.status == 401) {
      ctx.staus = 200;
      ctx.body = utils.fail('token 验证失败', utils.CODE.AUTH_ERROR);
    }else {
      throw err;
    }
  });
})

app.use(jwt({ secret: 'fang' }).unless({
  path: [/^\/api\/users\/login/]
}));
// ...

const router = require('koa-router')();
const User = require('./../models/userSchema');
const Util = require('../utils/utils');
var jwt = require('jsonwebtoken');
router.prefix('/users');

router.post('/login', async (ctx) => {
	try {
		const { userName, password } = ctx.request.body;
    console.log(userName, password);
    /* 
    从数据库中返回指定的字段有三种方式：
    1. 'userId userName userEmail state role deptId roleList'
    2.  {userName:1,_id:0} 这样可以 {userName:1,userPwd:0} 这样会报错  为什么？
    3. .select('userName userPwd')
    */
		const res = await User.findOne({
			userName,
			userPwd: password
      // 指定 mongoDB 返回哪些字段
		// }, 'userId userName userEmail state role deptId roleList');
		// }, {userName:1,_id:0});
		}).select('userId userName userEmail state role deptId roleList');

    const data = res._doc;
    const token = jwt.sign({
      data
      // 20 代表 20s
    }, 'fang', { expiresIn: 20 });

		if (res) {
      data.token = token;
			ctx.body = Util.success(data);
		} else {
			ctx.body = Util.fail('账号或密码不正确');

	} catch (err) {
