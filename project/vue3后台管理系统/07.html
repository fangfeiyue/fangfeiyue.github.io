表单 prop 的作用

  1. 问题：

用户管理界面引入分页组件后，分页组件的箭头并没有居中，而是偏向了左边。查看了原因了发现是箭头宽度的问题。

就在源码中找到了相关样式代码，添加了一行 width: inherit;

node_modules/element-plus/packages/theme-chalk/src/pagination.scss

// ...
.#{$namespace}-icon {
  display: block;
  font-size: 12px;
  font-weight: bold;
  // 规定应该从父元素继承 width 属性的值。
  width: inherit;
}
// ...

这里顺便说下 width:100%; 和 width:inherit; 的区别。 width:inherit; 是继承父元素的 width 值；width:100%; 是将元素的宽度设置为父元素的宽度。一般情况下，使用两者都可以，但是如果子元素脱离了文档流，这个时候使用 width:100%; 会有问题，所以在不考虑兼容性的情况下建议使用 width:inherit;（注意，IE8 不支持这个属性。）

export default {
	formateDate(date = new Date(date), rule) {
		let fmt = rule || 'yyyy-MM-dd hh:mm:ss';
		const o = {
			'y+': date.getFullYear(),
			'M+': date.getMonth(),
			'd+': date.getDate(),
			'h+': date.getHours(),
			'm+': date.getMinutes(),
			's+': date.getSeconds()
		};
		for (let k in o) {
			if (new RegExp(`(${k})`).test(fmt)) {
				const val = o[k] + '';
				fmt = fmt.replace(RegExp.$1, val > 9 ? val : '0' + val);
			}
		}
		return fmt;
	}
};

# 用户新增接口实现

功能：

在用户列表点击新增按钮，输入用户信息后，点击确定按钮，可以添加新用户。如果输入的用户名之前已经存在，需要提示用户已存在，请重新输入。

效果：

难点：

当我们新增用户的时候，需要用户 ID 自动增长，但 Mongodb 没法像 Mysql 一样自动增长 ID ，如果我们手动修改的话，那太麻烦了。这个时候我们可以新建一个 counter 表，专门用于用户 ID 的累加。

当新增用户的时候，先让这个 counter 表里的值增加，然后用这个增加后的值，当做这个新增用户的 ID，存入数据库。

在新建 counter 表后，要先给 counter 表中用于累加的值给定初始值，要不没法累加。这里我给定的是 2 ，因为我用户表中已经存在两个用户，所以新增的用户从 2 开始。
